01Läuft das auf meinem Gerät?

Nexoryx passt sich automatisch an. Drei Stufen, je nach Hardware:

Ultra-Lite CPU-only

Alte Laptops, ab ~4 GB RAM. Kleines lokales Modell (Ollama Tiny), sehr sparsam. Agenten laufen sequenziell.

Balanced 8 GB+

Normale Rechner. Mittlere Ollama-Modelle (7B), optional Cloud-Fallback. Begrenzte Parallelität.

Pro GPU

Starke GPU (≥24 GB VRAM) oder ≥48 GB RAM. Große Modelle (32–70B), Multi-Agent, volle Parallelität.

Nexoryx nutzt Ollama als lokale Inferenz-Engine und eigene Modelle als Spezialstufen. Alles wird beim Start automatisch erkannt — kein manuelles Konfigurieren nötig.

03API-Keys & Cloud-Provider

Alle Cloud-Keys sind optional — Nexoryx läuft vollständig offline ohne einen einzigen Key. Mit Keys routet der Model-Router automatisch in die Cloud, wenn lokale Modelle nicht ausreichen.

Provider Key-Name Woher Wofür
Anthropic empfohlen ANTHROPIC_API_KEY console.anthropic.com Claude — beste Qualität, Coding, Reasoning
OpenAI OPENAI_API_KEY platform.openai.com GPT-4o — Fallback, Vision, Embeddings
Google Gemini GOOGLE_API_KEY aistudio.google.com Gemini — Fallback, langer Kontext, Vision
A
Während der Installation
bootstrap.py fragt interaktiv nach jedem Key. Einfach eingeben oder leer lassen zum Überspringen.
B
Nachträglich per CLI
$ nexoryx admin key set anthropic sk-ant-…
C
Manuell (Umgebungsvariable)
$ export ANTHROPIC_API_KEY="sk-ant-…"
Sicherheit: Keys werden ausschließlich in ~/.nexoryx/secrets (chmod 600) gespeichert — nie im Repo, nie in Logs. Das öffentliche GitHub-Repo enthält zu keiner Zeit Credentials. Budget-Limits pro Provider sind in ~/.nexoryx/config.yaml konfigurierbar, um unerwartete Cloud-Kosten zu vermeiden.

04In 3 Schritten startklar

  1. Befehl kopieren (oben, je nach OS) und im Terminal / PowerShell als Admin einfügen.
  2. Fragen beantworten — Nexoryx prüft die Hardware, installiert Ollama-Modelle und fragt optional nach API-Keys & Telegram.
  3. Loslegen: nexoryx ask "Hallo" oder über den Telegram-Bot.

Ausführliche Schritt-für-Schritt-Anleitung pro Betriebssystem, inkl. API-Keys & Ollama → install.html

05Wichtige Befehle

nexoryx doctorHardware + Profil + Modelle prüfen

nexoryx run "…"Aufgabe über Planner + Agenten
nexoryx exec "…"Shell in Sandbox (Security + Audit)
nexoryx models pull tiny|mini|largeHardware-geprüfter Modell-Download
nexoryx memory …Speicher list/search/forget/remember

nexoryx telegramTelegram-Bot starten
nexoryx panicNot-Aus: alles stoppen
Admin-Funktionen nur über diesen Server. Wer Nexoryx über 192.168.13.100 installiert, wird Owner/Admin seines Geräts (Modelle, Keys, Rollen, Logs verwalten). Installationen aus dem öffentlichen Repo laufen im normalen User-Modus.
