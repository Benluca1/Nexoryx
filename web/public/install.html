01Voraussetzungen

Du brauchst nichts vorab zu installieren. Der Installer erledigt alles automatisch: Python 3.11+, Ollama, alle Abhängigkeiten. Einzige wirkliche Voraussetzung:

Linux / macOS curl muss vorhanden sein (bei macOS & fast allen Linux-Distros vorinstalliert)
Windows PowerShell 5.1+ (ab Windows 10 vorinstalliert) · oder WSL mit Ubuntu
Internet Für den Erst-Download von Python, Ollama und Modellen (einmalig)
RAM Mindestens 4 GB (läuft dann im Ultra-Lite-Modus mit Tiny-Modell)
Was der Installer automatisch einrichtet: Python 3.11 (falls nicht vorhanden), pip + venv, Nexoryx-Paket, Ollama (lokale Inferenz), passendes Startmodell je nach Hardware, ~/.nexoryx/-Konfigurationsverzeichnis, optionale Telegram-Integration. Auf Linux auch systemd-Service für automatischen Start.

02Installation — pro Betriebssystem

Linux

Terminal öffnen, Befehl einfügen, Enter drücken. Der Installer fragt interaktiv nach API-Keys und Telegram — alles kann übersprungen werden.

$ curl -fsSL http://192.168.13.100:3007/install.sh | bash

Installierts: Python 3.11 via apt/dnf/pacman (je nach Distro), Ollama, Nexoryx-Paket, systemd-Service.

macOS

Terminal öffnen (cmd+Space → „Terminal"), Befehl einfügen. Der Installer installiert Python via Homebrew falls nötig — Homebrew selbst wird ebenfalls automatisch installiert.

$ curl -fsSL http://192.168.13.100:3007/install.sh | bash

Installiert: Homebrew (falls fehlend) → Python 3.12 → Ollama → Nexoryx. Launchd-Service optional.

Windows

Option A — PowerShell (nativ, empfohlen): PowerShell als Administrator öffnen (Rechtsklick auf Startmenü → „Windows PowerShell (Administrator)"):

PS> irm http://192.168.13.100:3007/install.ps1 | iex

Installiert automatisch: Python 3.12 via winget, Ollama (Windows-Installer), Nexoryx-Paket, Windows-Dienst (optional).

Option B — WSL (Ubuntu): WSL installieren (wsl --install in Admin-PowerShell), dann Ubuntu starten:

$ curl -fsSL http://192.168.13.100:3007/install.sh | bash

Volle Linux-Kompatibilität. Empfohlen wenn du regelmäßig mit Linux-Tools arbeitest.

03API-Keys einrichten

Alle Keys sind vollständig optional. Ohne Keys läuft Nexoryx rein lokal über Ollama. Mit Keys routet der Model-Router automatisch in die Cloud für bessere Qualität oder wenn das lokale Modell überlastet ist.

Welche Provider gibt es?

ProviderEnv-VariableKey holenEinsatz im Router
Anthropic (Claude) empfohlen ANTHROPIC_API_KEY console.anthropic.com Beste Qualität — Coding, Reasoning, komplexe Tasks
OpenAI (GPT-4o) OPENAI_API_KEY platform.openai.com Fallback, Vision, Embeddings
Google Gemini GOOGLE_API_KEY aistudio.google.com Fallback, langer Kontext (>1M Token), Vision

Wie einrichten?

A
Während der Installation (einfachste Methode)
bootstrap.py fragt interaktiv nach jedem Provider-Key. Key eintippen oder einfach Enter drücken zum Überspringen. Keys können jederzeit nachträglich gesetzt werden.
[nexoryx] Anthropic API-Key (Claude) — leer lassen = überspringen
> sk-ant-api03-…
✓ Anthropic-Key gespeichert in ~/.nexoryx/secrets
B
Nachträglich per CLI
$ nexoryx admin key set anthropic sk-ant-api03-…
$ nexoryx admin key set openai sk-proj-…
$ nexoryx admin key set google AIzaSy…
C
Manuell per Umgebungsvariable
Für kurzzeitige Tests oder CI — Keys in Shell setzen, Nexoryx liest sie automatisch:
$ export ANTHROPIC_API_KEY="sk-ant-…" # Linux/macOS
PS> $env:ANTHROPIC_API_KEY = "sk-ant-…" # Windows PowerShell
D
Keys prüfen & Budget setzen
$ nexoryx admin key list
Budget-Limits (z.B. max. $5/Monat pro Provider) in ~/.nexoryx/config.yaml unter router.budget_caps. Bei Überschreitung routet Nexoryx automatisch auf lokale Modelle zurück.
Sicherheit: Keys werden ausschließlich in ~/.nexoryx/secrets (Dateiberechtigung 600) gespeichert. Das öffentliche GitHub-Repo enthält zu keiner Zeit Credentials. Logs maskieren Keys automatisch (sk-ant-***).

04Ollama-Modelle

Ollama ist die lokale Inferenz-Engine von Nexoryx. Der Installer richtet Ollama automatisch ein und lädt das passende Startmodell für deine Hardware. Du kannst jederzeit weitere Modelle hinzufügen.

Ollama manuell installieren (falls nicht automatisch)

Linux / macOS

$ curl -fsSL https://ollama.com/install.sh | sh

Windows

Ollama-Windows-Installer herunterladen und ausführen — danach läuft Ollama als Windows-Dienst:

PS> winget install Ollama.Ollama

Nexoryx — Modell-Befehle

nexoryx models list Alle verfügbaren & installierten Modelle anzeigen (lokal + Cloud)
nexoryx models pull tiny Nexoryx-Tiny (Control Brain) laden — hardware-geprüft
nexoryx models pull mini Nexoryx-Mini laden (ab Balanced-Profil)
nexoryx models pull large Nexoryx-Large laden (nur auf Pro-Hardware, ~20 GB)
nexoryx ask "…" --model ollama/qwen2.5:7b Bestimmtes Ollama-Modell erzwingen (ohne Router-Logik)
Automatische Erkennung: Wenn Ollama auf Port 11434 (Standard) läuft, erkennt Nexoryx es beim Start automatisch — kein manuelles Konfigurieren. Laufende Modelle werden in der Modell-Registry eingetragen und stehen im Router zur Verfügung. Prüfen mit: nexoryx doctor.

05Prüfen

$ nexoryx doctor

Zeigt: erkannte Hardware, gewähltes Profil (ultra_lite / balanced / pro), installierte Ollama-Modelle, gesetzte API-Keys (maskiert), Router-Status, Daemon-Status.

$ nexoryx ask "Erkläre mir kurz, was du bist."

Erster echter Test — der Router wählt automatisch das beste verfügbare Modell. In den Logs siehst du welches Modell gewählt wurde und warum.

06Telegram (optional)

Nexoryx kann vollständig über Telegram gesteuert werden — gleiche Befehle wie im Terminal, plus Push-Benachrichtigungen, Datei-Upload und Approve/Deny für riskante Aktionen.

  1. Bot bei @BotFather anlegen → /newbot → Token kopieren
  2. Eigene Telegram-User-ID herausfinden: @userinfobot schreiben
  3. Während Installation angeben — oder nachträglich:
    $ nexoryx telegram setup
  4. Bot starten: nexoryx telegram start — oder automatisch über den Daemon

Ohne Telegram: bei der Abfrage einfach Enter drücken (überspringen) oder Flag --no-telegram beim Bootstrap-Aufruf.

07Hilfe & Troubleshooting

Python nicht gefunden Der Installer installiert Python automatisch. Falls manuell nötig: Ubuntu → sudo apt install python3.12, macOS → brew install python@3.12, Windows → winget install Python.Python.3.12
Ollama antwortet nicht Status prüfen: ollama list. Dienst starten: ollama serve (Linux/macOS) oder Ollama-App öffnen (Windows). nexoryx doctor zeigt den Verbindungsstatus.
Modell zu groß / langsam Nexoryx wählt automatisch das kleinste passende Modell für dein Profil. Alternativ: kleineres Ollama-Modell ziehen, z.B. ollama pull llama3.2:3b.
API-Key-Fehler Key prüfen: nexoryx admin key list. Neu setzen: nexoryx admin key set anthropic <KEY>. Nexoryx fällt automatisch auf lokale Modelle zurück wenn ein Key fehlt/ungültig ist.
PowerShell: Script-Ausführung blockiert Policy kurz setzen: Set-ExecutionPolicy RemoteSigned -Scope CurrentUser (als Admin), danach nochmal versuchen.
Logs & Diagnose Alle Logs unter ~/.nexoryx/logs/. Echtzeit-Log: nexoryx logs tail.
Hinweis: Installation über 192.168.13.100 = Admin-Modus (Modelle, Keys, Rollen, Logs verwalten). Aus dem öffentlichen Repo installiert = User-Modus (ohne Admin-Funktionen).
